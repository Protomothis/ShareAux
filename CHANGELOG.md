## 0.1.4 (2026-04-19)

### 기능 추가

- 즐겨찾기 — 곡 하트 버튼으로 즐겨찾기 등록 (유저당 최대 200곡, 게스트 제외)
- 즐겨찾기 폴더 — 색상 dot 15색 프리셋, 유저당 최대 20개, 폴더명 2~20자
- 즐겨찾기 탭 — 검색 모달에 "추천 / 검색 / 즐겨찾기" 3탭 구성
- 폴더별 곡 섹션 표시 + 접기/펼치기
- 곡 드래그앤드롭으로 폴더 간 이동 (모바일 터치 지원)
- 편집 모드 — 다중 선택 후 일괄 폴더 이동 / 즐겨찾기 해제
- 폴더 관리 모달 — 생성/편집(이름+색상)/삭제
- 큐 아이템 + 재생 히스토리에 즐겨찾기 하트 표시
- 재생 히스토리에 좋아요/싫어요 버튼 추가
- MusicBrainz 폴백 — Content ID 없는 곡도 메타데이터 자동 매칭
- 가사 유형 구분 — LyricsType enum (synced/karaoke), 플레이어에 LRC/KLRC 표시
- 관리자 트랙 상세 — 가사 미리보기 (원문/번역 탭), 가사/Content ID 초기화, 트랙 삭제
- 관리자 트랙 목록 — 전체 트랙 조회 (재생 기록 없는 곡 포함), 번역 여부 표시

### 개선

- 가사 검색 워싱 강화 — CJK 괄호, 레이블명, feat. 제거, 노이즈 패턴 대폭 확장
- 가사 저장 시 언어 자동 감지 (이전엔 번역 시점에만 감지)
- 동시 요청 제어 — 가사 검색 최대 3개 동시, MusicBrainz 순차 큐
- 워싱 로직 모듈화 — title-cleaner.ts 공용 유틸 (가사 검색 + MusicBrainz 공유)
- detectLang 공용 유틸 분리
- 미사용 트랙 자동 정리에서 즐겨찾기 트랙 제외
- 스킵 시 즉시 전환 (자연 종료만 2초 대기)
- SourceBuffer 에러 후 자동 복구 — MSE 재생성 + 정지 표시

### 버그 수정

## 0.1.3 (2026-04-19)

### 기능 추가

- 재신청 쿨다운 — 같은 곡 재신청 제한 (제한없음 / 30분 / 60분 / 90분 / 방 종료까지)
- 검색 시 신청 불가 곡 비활성화 + "재신청 불가" 표시
- 관리자 트랙 목록에 가사 상태/언어 표시
- 관리자 계정 삭제 기능
- 초대코드별 유저 목록 조회 + 게스트 일괄 삭제

### 개선

- 곡 신청 시 목록 즉시 반영 (간헐적 누락 수정)
- 곡 자연 전환 시 마지막까지 재생 후 다음 곡 시작 (2초 버퍼)
- 스킵은 즉시 전환 (대기 없음)
- 방 삭제 후에도 재생 통계 보존 — 일별 재생 수 차트 정상 표시
- 가사 검색 실패 시 24시간 후 자동 재검색
- 번역 실패 시 불필요한 클라이언트 새로고침 제거
- 현재 재생 중인 곡 재신청 차단
- 첫 곡 재생 시 오디오 안 나오는 문제 수정
- 스킵 실패 시 UI "넘기는중" 고착 수정

### 내부 변경

- 검색 시 DB 저장 제거 — 큐에 추가된 곡만 저장
- 트랙 식별자 `provider` + `sourceId` 체계로 전환 (향후 다중 소스 확장 대비)
- API 호출 orval 자동 생성 함수로 통일 — 수동 customFetch 호출 제거
- 강제 캐스팅(`as unknown as`) 전부 제거 — 서버 response DTO 추가
- Provider enum 서버 정의 → orval 자동 생성
- fetch 로직 `api/mutator.ts`로 통합 — `api-client.ts` 제거
- swagger.json 제거 — orval이 서버에서 직접 fetch

### ⚠️ 업데이트 안내 (v0.1.2 → v0.1.3)

DB 스키마가 변경되었습니다. **업데이트 전 반드시 백업**하세요.

```bash
# 1. 백업
docker compose exec db pg_dump -U spotiparty spotiparty > backup_0.1.2.sql

# 2. 마이그레이션 실행
docker compose exec -T db psql -U spotiparty < migrations/v0.1.2-to-v0.1.3.sql

# 3. 이미지 업데이트
docker compose pull && docker compose up -d
```

신규 설치는 마이그레이션 불필요 — 자동으로 최신 스키마가 생성됩니다.

- 멤버 액션 메뉴: 클릭 → 권한 관리 / 신고하기 선택
- 신고 기능: ReportModal (사유 선택 + 상세 입력 + API 연동)
- WS 연결 끊김 배너: motion 애니메이션, 재연결 시 자동 사라짐
- 플레이어 썸네일 shimmer: 로딩/전환 시 빛 지나가는 효과
- 플레이어 텍스트 shimmer: 준비중/스킵 중 상태 텍스트

### 버그 수정

- 오디오 곡 전환: prepareResync()로 clearBuffer+gotInit 리셋 통합, stall 3초 자동 resync
- 프로그레스바 점프: streaming 전환 시 서버 elapsedMs 무조건 동기화
- 프로그레스바 드르륵: skipping/preparing 시 elapsed 즉시 리셋, isPlaying=false → idle
- audio.currentTime 보정: streaming일 때만 실행 (이전 곡 위치 오염 방지)
- 큐 애니메이션: seenIdsRef로 React 19 호환, overflow-x-hidden
- 검색 모달 footer: 선택 바 전체 너비 확보
- 게스트 로그인 후 authStore.init() 누락
- 게스트 프로필: 비밀번호/Google연동/탈퇴 숨김
- 방 설정 레이아웃: 신청횟수 초기화/제재 관리 여백 추가

### 개선

- useRoomEvents: onSystem if/else 체인 → 역할별 핸들러 분리
- useAudioControl: elapsedBase 변경 즉시 반영
- 에러 toast 중복 제거 (11개 개별 toast → 글로벌 1곳)
- 신고 관리 페이지: 처리/무시 설명 추가
